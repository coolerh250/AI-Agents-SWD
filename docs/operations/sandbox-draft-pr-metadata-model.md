# Sandbox GitHub — Draft PR Metadata Model (Step 59)

- Model: `infra/github/sandbox-draft-pr-metadata-model.yaml`
- SDK: `shared/sdk/sandbox_github/pr_metadata.py`
- Verifier: `scripts/verify_sandbox_github_pr_metadata.py` → `SANDBOX_GITHUB_PR_METADATA_VERIFY`

## Title
```
[Sandbox][Draft] {project_key}: {work_item_title}
```
The PR is always created with `draft: true` and never `ready_for_review`.

## Body — required sections
`Project`, `Work item`, `Dispatch`, `Correlation ID`, `Summary`, `Safety scope`,
`Non-production caveat`, `No production approval`, `No merge authorization`,
`Generated by AI Agents sandbox flow`.

## Body — forbidden content
`secret`, `token`, `credential`, `raw prompt`, `chain-of-thought`,
`production approval claim`, `customer data`. `validate_body` raises if a forbidden
string appears or a required section is missing.
